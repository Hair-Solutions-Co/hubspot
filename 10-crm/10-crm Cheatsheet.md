# 10-crm Cheatsheet

## The Model

| What | Where | Direction |
|------|-------|-----------|
| Property definitions | `10-crm/schemas/` | **Local → HubSpot** (you own this) |
| Pipeline / stage IDs | `config/id_manifest.json` | Auto-generated, commit it |
| Daily record counts | `10-crm/imports-exports/snapshots/YYYY-MM-DD/counts.json` | **HubSpot → Local** (pull only) |
| Full record exports | `10-crm/imports-exports/exports/` | On-demand only, don't commit |

---

## Commands

```bash
# Pull all property definitions + regenerate id_manifest.json
npm run hubspot:crm:schema

# Pull today's record counts (all objects + pipeline breakdowns)
npm run hubspot:crm:snapshot

# One-off: snapshot a single object (legacy, per-object CSV)
npm run hubspot:snapshot contacts

# On-demand: full record export for bulk work
npm run hubspot:exportcrm contacts

# On-demand: export property definitions for an object
npm run hubspot:exportprop deals
```

---

## When to Run What

| Trigger | Command |
|---------|---------|
| First time setup | `hubspot:crm:schema` then `hubspot:crm:snapshot` |
| After adding/changing any property | `hubspot:crm:schema` → commit output |
| Every morning (cron) | `hubspot:crm:snapshot` → commit output |
| Need to do bulk edits on records | `hubspot:exportcrm <object>` → edit CSV → re-import via HubSpot UI |
| Agent needs pipeline/stage IDs | Read `config/id_manifest.json` |
| Agent asks "how many X do we have" | Read `snapshots/YYYY-MM-DD/counts.json` |

```
10-crm/schemas/

  companies/
    properties.json

    properties.json

  tickets/
    properties.json
    pipelines.json
  orders/
    properties.json
    pipelines.json
  leads/ products/ quotes/ subscriptions/ invoices/ line_items/
    properties.json
  2-XXXXXXX/            ← custom objects use their objectTypeId as folder name
    properties.json
```

## `config/id_manifest.json` — What's In It

Auto-generated by `hubspot:crm:schema`. Agents read this to resolve names → IDs.

```json
{
  "portal": { "HUBSPOT_ACCOUNT__PRIMARY__PORTAL_ID": "50966981" },
  "custom_objects": {
    "My Object": { "objectTypeId": "2-12345678", "aliases": [...] }
  },
  "pipelines": {
    "deals": [
      { "id": "default", "label": "Sales Pipeline", "stages": { "Appointment Scheduled": "appointmentscheduled", ... } }
    ],
    "tickets": [ ... ]
  }
}
```

**Never edit this by hand.** Re-run `hubspot:crm:schema` and commit.

## Auth

All scripts run through `scripts/op_run.sh` which injects your service key from `.env` via 1Password. The canonical env var is `HUBSPOT_PRIVATE_APP__OPS__ACCESS_TOKEN`. If your `.env` uses `HUBSPOT_SERVICE_KEY`, `op_run.sh` remaps it automatically.

If you get an auth error:
```bash
# Check 1Password is signed in
op account list

# Manually test token injection
bash ./scripts/op_run.sh python3 -c "import os; print(bool(os.environ.get('HUBSPOT_PRIVATE_APP__OPS__ACCESS_TOKEN')))"
```

## What NOT to Do

- Don't commit full record CSVs — they go stale and are heavy
- Don't edit `id_manifest.json` by hand — re-run the schema pull
- Don't try to push records via script — use HubSpot UI imports for bulk record changes
- Don't store subscription preference changes locally — too risky to push via API
